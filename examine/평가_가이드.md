# KLSBench 전문가 평가 가이드

**대상**: 고려대학교 한문학과 연구원
**버전**: 1.0
**일자**: 2025-10-30

---

## 1. 평가 개요

### 목적
한국 고전 문학 스타일 벤치마크(KLSBench)의 학술적 품질을 검증하고, AI 모델 평가에 사용하기 전에 데이터의 정확성을 확보하기 위함입니다.

### 평가 범위
- **전체 데이터**: 7,871개 항목
- **평가 대상**: 780개 항목 (전체의 10% 샘플)
- **5개 태스크**: Classification, Retrieval, Punctuation, NLI, Translation

### 소요 시간 및 보상
- **예상 시간**: 2-3시간
- **사례금**: 10만원
- **평가 방식**: Excel 파일 작성 또는 온라인 폼

---

## 2. 태스크별 평가 방법

### Task 1: Classification (문체 분류)

#### 무엇을 평가하나요?
고전 한문 텍스트가 올바른 문체 장르로 분류되었는지 평가합니다.

#### 19개 분류 라벨
```
賦(부)      - 운문, 화려한 수사
詩(시)      - 정형시
疑(의문)    - 질의응답 형식
義(의론)    - 의리 논변
策(책문)    - 대책 논의
表(표문)    - 상소문
論(논)      - 논설문
記(기)      - 기록문
序(서)      - 서문
說(설)      - 설명문
箴(잠)      - 경계의 글
銘(명)      - 명문
頌(송)      - 송가
贊(찬)      - 찬사
哀辭(애사)  - 애도문
祭文(제문)  - 제사 지문
碑(비)      - 비문
禮義(예의)  - 예법 해설
經義(경의)  - 경전 해석
```

#### 평가 기준
1. **정확함**: 텍스트의 형식, 내용, 수사법이 라벨과 완전히 일치
2. **부분 정확**: 대체로 맞으나 다른 해석도 가능 (예: 論과 說의 경계)
3. **부정확함**: 라벨이 명백히 잘못됨

#### 예시

**예시 1: 정확한 경우**
```
원문: "惟天地之大德曰生 聖人之大寶曰位..."
라벨: 論
평가: ✓ 정확 (논설 형식, 주장-논거 구조 명확)
```

**예시 2: 부정확한 경우**
```
원문: "維歲次丙申 某月某日 謹以淸酌庶羞..."
라벨: 記
평가: ✗ 부정확 → 올바른 라벨: 祭文 (제사 지문의 전형적 형식)
```

#### 난이도 판단
- **1점**: 명백한 특징 (예: "謹以淸酌" → 祭文)
- **3점**: 전문 지식 필요 (논설의 종류 구분)
- **5점**: 전문가도 판단 곤란 (論/說/疑의 미묘한 차이)

### Task 2: Retrieval (정보 검색)

#### 무엇을 평가하나요?
주어진 쿼리(질문)에 대해 검색된 문서가 실제로 관련이 있는지 평가합니다.

#### 평가 기준
1. **매우 관련 있음**: 정답으로 적합, 직접적 연결
2. **관련 있음**: 부분적으로 연결됨
3. **약간 관련 있음**: 간접적 연결
4. **관련 없음**: 오답

#### 예시

**예시 1: 매우 관련 있음**
```
Query: "맹자가 말한 사단(四端)은 무엇인가?"
Document: "惻隱之心 仁之端也 羞惡之心 義之端也..." (측은지심은 인의 단서...)
평가: ✓ 매우 관련 (직접적 정답)
```

**예시 2: 관련 없음**
```
Query: "공자의 제자 중 덕행이 뛰어난 사람은?"
Document: "孟子曰 道性善..." (맹자가 말하길 성선설...)
평가: ✗ 관련 없음 (주제가 다름 - 공자 vs 맹자, 제자 vs 성리학)
```

#### 관련 없는 이유 체크리스트
- [ ] 주제가 다름
- [ ] 시대가 맞지 않음
- [ ] 인물/사건이 다름
- [ ] 문맥이 부적절함

---

### Task 3: Punctuation (구두점 삽입)

#### 무엇을 평가하나요?
구두점이 없는 한문 텍스트에 적절하게 구두점(、。)이 삽입되었는지 평가합니다.

#### 기본 규칙
- **、(반점)**: 구(phrase) 단위 구분, 짧은 호흡
- **。(마침표)**: 문장(sentence) 단위 구분

#### 평가 기준
1. **완전 정확**: 모든 구두점 위치가 정확
2. **대체로 정확**: 1-2개 위치에 논란 가능
3. **부분 정확**: 여러 곳에 오류
4. **부정확**: 대부분 잘못됨

#### 예시

**예시 1: 정확한 경우**
```
원문: 天命之謂性率性之謂道修道之謂教
정답: 天命之謂性、率性之謂道、修道之謂教。
평가: ✓ 완전 정확 (3개 병렬 구조를 、로 구분, 문장 끝 。)
```

**예시 2: 오류 있는 경우**
```
원문: 惻隱之心仁之端也羞惡之心義之端也
제시: 惻隱之心仁之端也。羞惡之心義之端也。
평가: ✗ 부분 정확
수정: 惻隱之心、仁之端也。羞惡之心、義之端也。
이유: 주어와 술어 사이에 、필요
```

#### 오류 유형
- **문장 경계 오류**: 。위치가 잘못됨
- **구 경계 오류**: 、위치가 잘못됨
- **의미 단위 오류**: 문맥 이해 부족
- **문법 오류**: 주어/술어 구분 실패

---

### Task 4: NLI (자연어 추론)

#### 무엇을 평가하나요?
전제(Premise)와 가설(Hypothesis) 사이의 논리적 관계가 올바르게 판단되었는지 평가합니다.

#### 3가지 라벨
1. **Entailment (함의)**: 전제가 참이면 가설도 반드시 참
2. **Contradiction (모순)**: 전제와 가설이 서로 모순
3. **Neutral (중립)**: 전제만으로는 가설의 참/거짓 판단 불가

#### 평가 기준
- 논리적 관계가 명백한가?
- 다른 해석의 여지가 있는가?
- 배경 지식이 필요한가?

#### 예시

**예시 1: Entailment**
```
Premise: "맹자는 성선설을 주장했다"
Hypothesis: "맹자는 인간의 본성이 선하다고 믿었다"
라벨: Entailment
평가: ✓ 정확 (성선설 = 인간 본성이 선하다)
```

**예시 2: Contradiction**
```
Premise: "공자는 인(仁)을 최고 덕목으로 삼았다"
Hypothesis: "공자는 인(仁)보다 예(禮)를 더 중시했다"
라벨: Contradiction
평가: ✓ 정확 (상호 배타적 진술)
```

**예시 3: Neutral**
```
Premise: "이황은 조선시대 성리학자였다"
Hypothesis: "이황은 퇴계라는 호를 사용했다"
라벨: Neutral
평가: ✓ 정확 (전제만으로 가설 판단 불가 - 배경 지식 필요)
```

#### 주의사항
- 배경 지식을 사용해야 하는 경우 → Neutral
- 논리적 추론만으로 판단 가능한 경우 → Entailment/Contradiction

### Task 5: Translation (번역)

#### 무엇을 평가하나요?
한문 원문이 현대 한국어로 정확하고 자연스럽게 번역되었는지 평가합니다.

#### 평가 기준

##### 1) 정확성
- **완전 정확**: 원문 의미 완벽 전달
- **대체로 정확**: 미세한 뉘앙스 차이
- **부분 정확**: 일부 오역 존재
- **부정확**: 중대한 오역

##### 2) 자연스러움
- **매우 자연스러움**: 현대 한국어로 유창
- **자연스러움**: 일반적 표현
- **보통**: 이해 가능하나 어색
- **부자연스러움**: 직역 투
- **매우 부자연스러움**: 이해 곤란

#### 예시

**예시 1: 완전 정확 + 자연스러움**
```
원문: 學而時習之 不亦說乎
번역: 배우고 때때로 익히면 또한 기쁘지 아니한가
평가: ✓ 완전 정확, 자연스러움
```

**예시 2: 오역 있음**
```
원문: 君子不器
제시: 군자는 그릇이 아니다
평가: ✗ 부정확 (직역)
수정: 군자는 한 가지 용도에만 쓰이지 않는다
이유: "器"는 비유적 표현 (특정 용도의 도구)
```

**예시 3: 부자연스러움**
```
원문: 道之不行 已知之矣
제시: 도의 행해지지 않음을 이미 알았다
평가: △ 대체로 정확하나 부자연스러움
수정: 도가 실현되지 않을 것을 이미 알았다
이유: 현대 한국어 어순과 표현 개선 필요
```

#### 오역 유형
- **어휘 오역**: 단어 선택 오류
- **문법 오역**: 구조 이해 오류
- **문맥 오역**: 맥락 이해 부족
- **전거 오역**: 고사성어/인용 오류
- **생략/추가**: 원문에 없는 내용

## 3. 평가 진행 방법

### Step 1: 파일 받기
- Excel 파일: `전문가평가_템플릿.xlsx`
- 5개 시트: Classification, Retrieval, Punctuation, NLI, Translation

### Step 2: 각 항목 평가
각 시트에서:
1. 원문 읽기
2. 현재 라벨/정답 확인
3. 정확성 판단 (정확함 / 부분 정확 / 부정확함)
4. 오류 있으면 올바른 답 제시
5. 난이도 평가 (1-5점)
6. 추가 의견 작성 (선택)

### Step 3: 전반적 평가
마지막 시트에서:
- 각 태스크의 전반적 품질 평가
- 개선 필요 사항 기술
- 벤치마크 전체에 대한 의견

### Step 4: 제출
- 이메일 또는 제공된 플랫폼에 업로드
- 문의사항은 담당자에게 연락

## 4. 평가 팁

### 시간 관리
- **Classification**: 약 30분 (80개)
- **Retrieval**: 약 40분 (120개)
- **Punctuation**: 약 50분 (200개)
- **NLI**: 약 40분 (180개)
- **Translation**: 약 50분 (200개)
- **전반 평가**: 약 10분

**팁**: 명백히 정확한 항목은 빠르게 넘어가고, 논란이 있는 항목에 시간 투자

### 평가 일관성 유지
- 처음 30개 정도는 천천히 평가하여 기준 잡기
- 애매한 경우 "부분 정확" 선택
- 의견란에 판단 근거 간단히 작성

### 휴식
- 50개마다 5분 휴식 권장
- 한 번에 다 하려 하지 말고 2-3일에 걸쳐 진행 가능

## 5. 자주 묻는 질문 (FAQ)

### Q1: 애매한 경우는 어떻게 평가하나요?
**A**: "부분 정확"을 선택하고, 의견란에 어떤 점이 애매한지 간단히 적어주세요. 예: "論과 說의 경계가 모호함"

### Q2: 모든 항목에 의견을 작성해야 하나요?
**A**: 아니요. 오류가 있거나 특별히 언급하고 싶은 경우만 작성하시면 됩니다.

### Q3: 한문 원문 독해가 어려운 경우는?
**A**: 난이도를 높게 평가하고, 가능한 범위에서 판단해 주세요. 완전히 이해가 안 되는 경우 의견란에 표시해 주세요.

### Q4: 여러 개의 오류가 있는 경우는?
**A**: 가장 중요한 오류 위주로 지적하고, 의견란에 추가 오류 나열해 주세요.

### Q5: 평가 도중 문의사항이 생기면?
**A**: 언제든지 담당자에게 연락 주세요.
- 이메일: [담당자 이메일]
- 전화: [담당자 전화번호]
